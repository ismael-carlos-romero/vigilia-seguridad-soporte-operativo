# Soporte Operativo Interno de Vigilia Seguridad

## Manual breve de uso

### Objetivo del sistema

Soporte Operativo Interno de Vigilia Seguridad es una herramienta interna para acompañar el trabajo de la Central de Monitoreo. Su objetivo es reducir dudas operativas, ordenar procedimientos, derivar consultas al sector correcto y transformar cada evento irregular en conocimiento reutilizable.

El sistema busca que la informacion no quede dispersa en WhatsApp, mails, hojas impresas o memoria individual. Cada consulta, directiva, duda y resolucion puede quedar documentada, medida y disponible para otros operadores.

### Para que sirve

- Guiar al operador durante llamadas entrantes y eventos de monitoreo.
- Clasificar consultas por sector: Objetivo Fijo 911, SAFE, supervision, tecnica, administracion u otros.
- Consultar procedimientos operativos y tecnicos.
- Registrar eventos en espera o dudas que no se pueden cedular con seguridad.
- Estandarizar notas para enviar a supervisores o encargados.
- Convertir respuestas validadas por supervision en nuevos procedimientos.
- Medir eventos, dudas, tareas, turnos y oportunidades de mejora.

### Flujo recomendado

1. Ingresar al sistema con usuario y contrasena.
2. Verificar que aparezcan nombre, turno, sector y estacion.
3. En la seccion Ingreso, escribir la consulta del cliente tal como la expresa.
4. Validar cuenta, abonado, titular, palabra clave o dato adicional autorizado.
5. Seguir la recomendacion del sistema y abrir la seccion correspondiente.
6. Aplicar el procedimiento indicado.
7. Si se resuelve, registrar o cedular segun corresponda.
8. Si no se resuelve o queda duda, cargar el caso en Aprendizajes.
9. Copiar la nota estandarizada para consultar a supervision.
10. Cuando supervision valida el criterio, cargar la resolucion para convertirla en procedimiento reutilizable.

### Secciones principales

**Ingreso**
Clasifica una llamada o consulta. Sugiere sector, prioridad, preguntas guia y accion inicial.

**Objetivo Fijo 911**
Contiene guias para alarmas domiciliarias, teclados, fallas, aplicaciones, paneles y soporte operativo.

**Objetivos Moviles SAFE**
Contiene guias para monitoreo vehicular, unidades sin reporte, eventos GPS, desvios, panico, paradas y seguimiento.

**Procedimientos**
Base de eventos operativos recibidos en Bykom. Muestra chequeos, decisiones, acciones recomendadas y notas sugeridas.

**Supervision**
Espacio para cargar directivas, reglas o instrucciones que llegan desde encargados o supervisores.

**Aprendizajes**
Registro de eventos en espera, dudas, irregularidades y casos que deben validarse. Es la base de mejora continua.

**Kanban**
Tablero visual para ordenar tareas del turno, seguimientos, dudas y eventos pendientes.

**Metricas**
Muestra informacion para medir productividad real: eventos, estados, operadores, turnos, dudas, Pareto y oportunidades de mejora.

**Casos**
Historial de registros operativos.

### Roles

**Operador**
Usa el sistema durante el turno, consulta procedimientos, registra dudas y alimenta la base de conocimiento.

**Supervisor o encargado**
Valida eventos dudosos, define el criterio correcto y transforma respuestas en procedimientos.

**Administrador**
Gestiona usuarios, permisos, estructura del sistema, calidad de datos y evolucion de la herramienta.

### Como se convierte una duda en procedimiento

Cuando un operador encuentra un evento que no puede resolver con seguridad, lo carga en Aprendizajes con abonado, falla, duda concreta y contexto. El sistema genera una sugerencia y permite copiar una nota estandarizada para enviar a supervision.

Luego, el supervisor revisa el caso, confirma el criterio correcto y carga la resolucion: causa, procedimiento, nota para Bykom, derivacion y palabras clave. Esa informacion queda disponible para futuras busquedas, evitando repetir la misma consulta.

### Buenas practicas de carga

- Usar siempre abonado u objetivo, por ejemplo: `SZ-4578 - Iguatemi`.
- Escribir la falla de forma clara: `tamper reiterado`, `falla comunicacion`, `zona 10 dispara`.
- Separar duda concreta de contexto.
- Registrar que se intento antes de consultar.
- Cargar solo informacion util para resolver el caso.
- Evitar datos sensibles innecesarios.
- Si se envia una consulta por mail, usar el boton de nota estandarizada.

### Guion breve para video de presentacion

Soporte Operativo Interno de Vigilia Seguridad es un sistema interno creado para acompanar al operador en la Central de Monitoreo.

Su objetivo es reducir dudas, ordenar procedimientos y transformar cada evento irregular en conocimiento reutilizable.

El operador ingresa una consulta o llamada, el sistema sugiere el sector correspondiente, muestra preguntas guia y orienta la accion inicial.

Si el caso pertenece a Objetivo Fijo 911, SAFE, supervision o servicio tecnico, el sistema permite avanzar hacia la seccion correcta y consultar el procedimiento.

Cuando aparece una duda que no se puede resolver en el momento, se registra en Aprendizajes. Luego, supervision valida el criterio correcto y esa respuesta queda disponible para futuros casos.

Ademas, el tablero Kanban y las metricas permiten medir eventos, dudas, tareas, turnos y oportunidades de mejora.

La finalidad es profesionalizar la operacion, disminuir cuellos de botella y construir una base de conocimiento viva para toda la empresa.
