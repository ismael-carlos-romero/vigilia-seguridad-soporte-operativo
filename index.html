Soporte Operativo Interno de Vigilia Seguridad

Este sistema ordena la atención operativa, reduce dudas repetidas y convierte cada evento irregular en conocimiento reutilizable. Su objetivo es que el operador tenga una guía rápida para actuar, registrar, derivar y aprender sin depender siempre de instrucciones sueltas por WhatsApp, mail o memoria.

Objetivo

Para qué sirve

Centraliza procedimientos, directivas, eventos en espera, tareas y métricas para que la Central de Monitoreo trabaje con criterios claros, medibles y mejorables.

  • Guiar al operador durante llamadas y eventos.
  • Derivar consultas al sector correcto.
  • Consultar procedimientos de Objetivo Fijo 911 y SAFE.
  • Registrar dudas que necesitan validación de supervisión.
  • Convertir respuestas de supervisores en nuevos procedimientos.

Uso diario

Flujo recomendado

  1. Ingresar al turno: iniciar sesión para que el sistema registre operador, turno, sector y estación.
  2. Recibir consulta: cargar lo que dice el cliente en la sección Ingreso.
  3. Validar cuenta: confirmar abonado, titular, palabra clave o dato adicional y revisar Bykom cuando corresponda.
  4. Seguir guía: abrir la sección sugerida por el sistema y aplicar procedimiento.
  5. Cerrar o escalar: si se resuelve, registrar; si queda duda, guardarla en Aprendizajes.

Secciones

Qué hace cada módulo

  • Ingreso: clasifica llamadas y sugiere sector, preguntas y acción inicial.
  • Objetivo Fijo (911): guía para fallas, teclados, paneles, apps y soporte domiciliario.
  • Objetivos Móviles (SAFE): guía para unidades, camiones, GPS, sin reporte y seguimiento vehicular.
  • Procedimientos: base operativa para eventos recibidos en Bykom.
  • Pedidos ATCL: seguimiento trazable de pedidos de clientes con responsable, estado, evidencia y alertas.
  • Supervisión: directivas internas cargadas por responsables.
  • Aprendizajes: eventos en espera, dudas y casos que deben validarse.
  • Sugerencias: espacio para que cada usuario proponga mejoras del sistema, procesos, comunicación o capacitación.
  • Kanban: tablero visual para ordenar tareas y seguimientos.
  • Métricas: Pareto, eventos por operador, turno, estado y aprendizaje.

Roles

Responsabilidades

  • Operador: consulta, ejecuta procedimientos, registra eventos en espera y alimenta el sistema.
  • Supervisor / encargado: valida dudas, define criterios correctos y convierte casos en procedimientos.
  • Administrador: gestiona usuarios, permisos, estructura del sistema y calidad de la base de conocimiento.

Permisos

Niveles de usuario

  • Nivel 3 - Operador: usa Ingreso, 911, SAFE, Procedimientos, Aprendizajes, Sugerencias, Kanban y Métricas propias.
  • Nivel 6 - Técnica: consulta recursos técnicos, materiales de apoyo y eventos derivados a servicio técnico.
  • Nivel 8 - Supervisor: revisa eventos de operadores, resuelve dudas y valida criterios operativos.
  • Nivel 9 - Gerencia: ve métricas globales, seguimiento por sector y prioridades de mejora.
  • Nivel 10 - Administrador: administra usuarios, permisos, recursos sensibles y configuración general.

Alta de usuarios

Cómo crear usuarios y contraseñas

  1. Crear usuario en Firebase Authentication: cargar email y contraseña temporal.
  2. Copiar el UID: Firebase genera un identificador único para ese usuario.
  3. Crear perfil en Firestore: en la colección usuarios, crear un documento usando ese UID.
  4. Cargar campos obligatorios: name, email, role, level, sector, shift y active.
  5. Probar ingreso: entrar a la web con ese usuario y verificar que muestre nombre, nivel, turno, sector y estación.

La planilla de base está en usuarios_vigilia_firebase.csv. Las contraseñas son temporales y conviene cambiarlas cuando el sistema pase a uso real.

Trazabilidad

Gestión de pedidos ATCL

El sistema no reemplaza ATCL, WhatsApp ni el mail. Su función es centralizar el seguimiento: cada pedido del cliente queda registrado, asignado a un área responsable, medido por fecha y estado, y cerrado con observación o evidencia para que no dependa de pedidos verbales o memoria.

Cómo una duda se convierte en procedimiento

Cuando un operador encuentra un evento que no puede cedular con seguridad, lo carga en Aprendizajes con abonado, falla, duda concreta y contexto. El sistema sugiere una posible solución, permite copiar una nota estandarizada para supervisión y guarda el caso. Luego, el supervisor valida el criterio correcto y esa respuesta queda asentada como procedimiento reutilizable para otros operadores.

Presentación sugerida

Soporte Operativo Interno de Vigilia Seguridad es un sistema interno creado para acompañar al operador en la Central de Monitoreo.

Su objetivo es reducir dudas, ordenar procedimientos y transformar cada evento irregular en conocimiento reutilizable.

El operador ingresa una consulta o llamada, el sistema sugiere el sector correspondiente, muestra preguntas guía y orienta la acción inicial.

Si el caso pertenece a Objetivo Fijo 911, SAFE, supervisión o servicio técnico, el sistema permite avanzar hacia la sección correcta y consultar el procedimiento.

Cuando aparece una duda que no se puede resolver en el momento, se registra en Aprendizajes. Luego, supervisión valida el criterio correcto y esa respuesta queda disponible para futuros casos.

Además, el tablero Kanban y las métricas permiten medir eventos, dudas, tareas, turnos y oportunidades de mejora.

La finalidad es profesionalizar la operación, disminuir cuellos de botella y construir una base de conocimiento viva para toda la empresa.

Trazabilidad operativa

Gestión y seguimiento de pedidos ATCL

Registrá cada pedido del cliente con responsable, estado y evidencia. La idea es que nada quede solo verbal, por WhatsApp o perdido en un mail.

Criterio: el sistema no reemplaza ATCL; asegura que cada pedido quede centralizado, asignado, medible y verificable.

Cada pedido queda asociado al operador, turno, sector y estación.
